Custome Directive
constructor(private _renderer: Renderer2, private ref: ElementRef ) {}

@HostBinding('attr.role') role = 'admin';

@HostListener('click') onClickFun(){
 this.role = this.role == 'admin' ? 'guest' : 'admin';
}

@HostListener('mouseenter') mouseenterFun(){
 this._renderer.setStyle(this.ref.nativeElement,'backgroundColor', 'red')
}

@HostListener('mouseleave') mouseleaveFun(){
 this._renderer.setStyle(this.ref.nativeElement,'backgroundColor', 'yellow')
}




Custom Pipe
export class MultiplyPipe implements PipeTransform {
 transform(value: number, ...args: number[]): number {
  let output = 1;
  args.forEach((elm) => {
   output *= elm
  })
  return output * value;
 }
}

Metadata & Decorator

A Decorator is a special kind of declaration that can be attached to a class declaration, method, accessor, property, or parameter. Decorators use the form @expression, where expression must evaluate to a function that will be called at runtime with information about the decorated declaration.

Metadata is used to decorate a class so that it can configure the expected behavior of the class. Following are the different parts for metadata.

polyfills

it is basically those extra lines of code that makes your application compatible with different browsers. Generally, the code we write is in ES6 which is not compatible with IE or firefox and needs some background setups before being able to be viewed or used in these browsers. So, the polyfills take up the task and do these low-level setups for you.
